HOW TO WRITE A NEW RESPONSE READER
===================================

A response reader is a function that loads fMRI response data from disk and
returns it as a dict mapping run names to numpy arrays.

1. CREATE A NEW FILE in this directory (response_loaders/).
   Name it after your format, e.g. `my_lab_format.py`.

2. IMPORT the registration decorator:

       from denizenspipeline.plugins.response_loaders.readers import response_reader

3. WRITE a function with this exact signature and decorate it:

       @response_reader("my_reader_name")
       def _read_my_format(
           resp_dir: Path, run_names: list[str] | None, config: dict,
       ) -> dict[str, np.ndarray]:
           ...

   Arguments:
     - resp_dir (Path):  The directory or file from the YAML `response.path`
                         config. May be a directory or a file path.
     - run_names (list[str] | None):  If not None, load only these runs.
                                      If None, discover and load all runs.
     - config (dict):  The full `response:` section from the YAML config.
                       Use this to read any reader-specific keys you need
                       (e.g. config.get("my_custom_key", "default")).

   Return value:
     - dict[str, np.ndarray] mapping run_name -> array of shape (n_trs, n_voxels).
       Every array MUST be 2-D. If your raw data has extra dimensions
       (e.g. repetitions), collapse them before returning.

4. MAKE SURE the file is imported at startup. Add an import in
   the __init__.py of this package, OR import it from your YAML config
   script, OR use an entry point. The simplest approach:

       # In response_loaders/__init__.py, add:
       import denizenspipeline.plugins.response_loaders.my_lab_format  # noqa: F401

5. USE IT in your YAML config:

       response:
         loader: local
         reader: my_reader_name
         path: /path/to/data/
         my_custom_key: some_value    # any reader-specific keys go here

EXAMPLE — reference multiphase_hdf.py in this directory for a complete real reader.

TEMPLATE:

    """Reader for <your format description>."""
    from __future__ import annotations

    from pathlib import Path

    import numpy as np

    from denizenspipeline.plugins.response_loaders.readers import response_reader


    @response_reader("your_reader_name")
    def _read_your_format(
        resp_dir: Path, run_names: list[str] | None, config: dict,
    ) -> dict[str, np.ndarray]:
        # 1. Resolve the file(s) to open from resp_dir and config
        # 2. Open and read the data
        # 3. If run_names is not None, only load those runs
        #    If run_names is None, discover all available runs
        # 4. Return {run_name: array} where each array is (n_trs, n_voxels)

        responses: dict[str, np.ndarray] = {}

        # ... your loading logic here ...

        return responses

CHECKLIST:
  [ ] Function decorated with @response_reader("name")
  [ ] Signature: (Path, list[str] | None, dict) -> dict[str, np.ndarray]
  [ ] Handles run_names=None (discover all) and run_names=list (load subset)
  [ ] All returned arrays are 2-D: (n_trs, n_voxels)
  [ ] File is imported so the decorator runs at startup
  [ ] Reader-specific config keys documented in the module docstring
